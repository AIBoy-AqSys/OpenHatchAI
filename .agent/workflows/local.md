---
description: ローカルプロジェクト内での作業を行うためのワークフロー
---

1. `.agent/workflows/initialize.md` の手順に従い、ログファイルを作成・開始する。
2. 作業対象が「ローカルプロジェクト内」であることを再確認する。
3. マニフェストファイル（`README.md`、`AI_INSTRUCTIONS.md`、`INITIAL_MANIFEST/` 等）および `.env` など、プロジェクトルート配下のファイルのみを参照・変更する。
4. Shell コマンドを実行する場合は必ず相対パスを使用し、プロジェクトルート外へのアクセスや変更を行わない。
5. 作業が完了したら、ログに「作業完了」「変更内容の要約」を記載する。
6. .env や .env_sample 意外のファイルの新規作成や編集の場合、.env に記載してある値は .env の index でマスクしてください。
