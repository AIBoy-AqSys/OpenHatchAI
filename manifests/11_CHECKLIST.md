# 11_CHECKLIST.md（初回通し確認）
責務種別: `BUILD_VERIFICATION`

## 読み方（混在章の扱い）
- `BUILD + RUNTIME_POLICY` の章は、導入状態の確認（BUILD観点）と運用前提の確認（RUNTIME_POLICY観点）を分けて確認する
- 初回通し確認では「今すぐIssue解決まで完遂できるか」ではなく、「そのための基盤が揃っているか」を中心に確認する

## 何を行うのか
- VPS構築・GitHub設定・agentd稼働・Webゲームテンプレの4領域で初回の通し確認を行う
- 「導入済み」だけでなく、認証・enqueue・自律修正/検証ループ・PR作成・保存・CI通過などの動作確認まで実施する
- ブランチ作成前に `main` 最新化が行われることを確認する
- 自律実行中の進捗通知（開始/中間/完了/エラー）が期待どおり出ることを確認する
- ブラウザ検証が導入され、agentd から利用可能であることを確認する
- Provider切替（`codex` / `claude` / `gemini`）設定が読み取れることを確認する
- `12_SERVER_AI_INSTRUCTIONS.md` がサーバ側基盤指示書として配置されていることを確認する
- 追加強化項目（開始/完了コメント、デプロイ、メモリ制限等）が反映されているか確認する

## チェック対象カテゴリ
- A. VPS 基盤
- B. GitHub 連携
- C. agentd 実行フロー（`main` 最新化 / 自律修正・検証 / ブラウザ検証 / PR化 / Provider切替）
- D. Webゲームテンプレ稼働
- E. サーバ側基盤指示書の配置と参照性
