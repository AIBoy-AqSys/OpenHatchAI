# 02_SERVER_BOOTSTRAP.md（VPS初期構成）
責務種別: `BUILD`

## 何を行うのか
- 初回ログイン後のVPSを、AIエージェント実行基盤として最低限運用可能な状態にする
- root直運用を避け、専用運用ユーザー中心の構成に切り替える
- SSH・Firewall・依存導入・HTTPS 前提を整える
- GitHub連携と agentd 導入の前提を作る

## 実施項目（BUILD）
- OS更新を行う
- 低メモリ環境向けに swap を追加する
- 専用運用ユーザー（例: `agent`）を作成し、権限方針を定義する
- SSH の最低限ハードニング（rootログイン禁止、認証方針の固定）を行う
- Firewall を設定し、必要ポートのみ開放する
- 基本依存（Git / Python / Node / Webサーバ / gh CLI など）を導入する
- HTTPS化の前提（証明書取得・更新運用）を整える
- `agent` ユーザーで GitHub SSH 認証と gh 認証を有効化する
- ブラウザ検証基盤導入の前提（必要ライブラリ/実行環境）を整える
- `.env` 配置先と権限方針を決める

## 到達状態
- VPSが「常駐エージェントを動かせる最小構成」になっている
- Webhook受信・agentd常駐・GitHub連携・ブラウザ検証導入の前提が揃っている
