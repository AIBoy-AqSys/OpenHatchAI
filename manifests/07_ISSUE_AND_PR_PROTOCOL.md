# 07_ISSUE_AND_PR_PROTOCOL.md（指示・修正プロトコル）
責務種別: `RUNTIME_POLICY`

## 何を行うのか
- Issueコメントでエージェントを起動するための指示フォーマットを定義する
- 成功条件・制約・テスト観点を短く明示する運用ルールを決める
- 自律修正/検証ループの終了条件・失敗条件・報告条件を明文化する
- PRコメントでの追加修正指示も同じ型で統一する
- エージェントに渡す禁止事項を明文化する
- iPhone からでも運用しやすい最小操作セットを整理する
- 長い実行でも状況を追えるよう、進捗報告粒度を定義する
- Provider 指示（`codex` / `claude` / `gemini`）や browser検証必須の指定方法を定義する

## 指示フォーマットに含める要素（実装例は別途）
- 実行トリガ（起動指示）
- 目的 / 成功条件
- 制約 / 禁止事項
- テスト観点（コード検証 / ブラウザ検証）
- Provider指定（任意）
- 進捗報告要否
- 停止時の報告形式

## 目的
- 指示品質のばらつきを減らし、常駐エージェントの成功率を上げる
- 修正往復のコストを減らす
- 自律反復を前提にしても、人間が制御可能な運用を保つ
